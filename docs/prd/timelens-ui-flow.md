## 1. 공통 UI 레이아웃

### 1.1 기본 화면 구조

모바일 기준 (320px 뷰포트). 카메라가 항상 전체 배경.

```
┌──────────────────────────────┐
│  ● REC          🔋 📶  9:41 │ ← 상태바 (시스템)
├──────────────────────────────┤
│                              │
│                              │
│                              │
│      [ 카메라 라이브 뷰 ]     │ ← 전체 화면 배경
│       (항상 활성 상태)        │
│                              │
│                              │
│                              │
│                              │
├──────────────────────────────┤
│  🤖 Curator Agent            │ ← 에이전트 전환 표시기
├──────────────────────────────┤
│  ┌────────────────────────┐  │
│  │ ≋≋≋≋▐▌≋≋≋≋≋≋▐▌≋≋≋≋≋  │  │ ← 오디오 비주얼라이저 (파형)
│  └────────────────────────┘  │
│  ◉ 듣는 중...                │ ← 상태 표시 (대기/듣는중/말하는중)
├──────────────────────────────┤
│ ┌──────────────────────────┐ │
│ │                          │ │
│ │   Knowledge Panel        │ │ ← 하단 슬라이드업 패널
│ │   (슬라이드업 영역)       │ │    드래그로 높이 조절 가능
│ │                          │ │
│ └──────────────────────────┘ │
├──────────────────────────────┤
│  [🎤]     [📸]     [📖]    │ ← 하단 액션 바
│  음성      캡처     다이어리  │    (마이크/사진캡처/다이어리)
└──────────────────────────────┘
```

### 1.2 Knowledge Panel 상태

Knowledge Panel은 3단계 높이로 동작:

| 상태 | 높이 | 트리거 |
|---|---|---|
| **닫힘** | 0px (숨김) | 대기 상태, 아래로 스와이프 |
| **미니** | ~120px (요약 카드만) | 유물 인식 직후 자동 팝업 |
| **확장** | ~60% 화면 | 요약 카드 탭 또는 위로 스와이프 |
| **풀스크린** | ~90% 화면 | Before/After 슬라이더, 다이어리 |

### 1.3 에이전트 전환 표시기

```
┌──────────────────────────────┐
│  🤖 Curator → 🎨 Restoration │ ← 전환 중 (0.5초 애니메이션)
│  "복원 이미지를 생성합니다"    │
└──────────────────────────────┘
```

에이전트별 아이콘:
- 🤖 **Curator** — 해설/대화 (기본)
- 🎨 **Restoration** — 유물 복원
- 🧭 **Discovery** — 주변 탐색
- 📖 **Diary** — 다이어리 생성

### 1.4 오디오 비주얼라이저 상태

| 상태 | 표시 | 설명 |
|---|---|---|
| 대기 | `───────────────` (평평한 선) | 세션 연결됨, 입력 없음 |
| 듣는 중 | `≋≋≋▐▌≋≋≋▐▌≋≋≋` (사용자 파형) | 사용자 음성 입력 중 (파란색) |
| 말하는 중 | `∿∿∿∿∿∿∿∿∿∿∿∿∿` (AI 파형) | AI가 음성 출력 중 (보라색) |
| 생성 중 | `⠋ ⠙ ⠹ ⠸ ⠼ ⠴ ⠦ ⠧` (스피너) | 이미지 생성 등 대기 |

## 2. 시나리오 1: 박물관 — "이 도자기 뭐야?"

> **배경**: 사용자가 대영박물관에서 금 간 아테네 적회식 크라테르 앞에 서 있다.
> **목표**: 유물 인식 → 핵심 확인 → 토픽 탐색 → 음성 질문 → 복원 요청 → 다이어리 저장

### 2.1 화면 1: 카메라 뷰 (대기)

```
┌──────────────────────────────┐
│  ● REC          🔋 📶  9:41 │
├──────────────────────────────┤
│                              │
│                              │
│                              │
│    ┌────────────────────┐    │
│    │                    │    │
│    │   (박물관 진열장    │    │
│    │    라이브 카메라)   │    │
│    │                    │    │
│    └────────────────────┘    │
│                              │
│                              │
│                              │
│                              │
│  🤖 Curator Agent            │
│  ─────────────────────────── │
│  ◉ 카메라를 유물에 향해주세요  │
├──────────────────────────────┤
│  [🎤]     [📸]     [📖]    │
└──────────────────────────────┘
```

| 항목 | 내용 |
|---|---|
| **사용자 액션** | 앱을 열고 카메라 권한 허용. 진열장 쪽으로 카메라를 든다 |
| **시스템 반응** | WebSocket 연결 수립, 카메라 프레임 1fps 전송 시작, 상태: "카메라를 유물에 향해주세요" |
| **전환 힌트** | 카메라 프레임에 유물이 감지되면 → 화면 2로 전환 (페이드) |
| **에러 상태** | 카메라 권한 거부 → 권한 안내 오버레이 표시 |

| **타임아웃** | 5초 이상 인식 실패 → "인식이 어렵습니다. 더 가까이 비춰주세요" 안내 |

| 항목 | 내용 |
|---|---|

| **에러 상태** | 인식 결과 확신도 낮음 → "이 유물이 맞나요?" 확인 카드 표시 |

| 항목 | 내용 |
|---|---|
| **사용자 액션** | 토픽 칩 "용도"를 탭 |

| **전환 힌트** | "더 질문하기" 탭 → 화면 6 / 다른 토픽 칩 탭 → 해당 주제로 교체 / 패널 아래 스와이프 → 화면 3으로 복귀 |

## 부록: 전환 애니메이션 요약

| 전환 | 애니메이션 | 지속 시간 | 이징 |
|---|---|---|---|
| 대기 → 스캐닝 | 스캔 코너 페이드인 + 펄스 | 200ms in, 1s pulse loop | ease-out |
| 스캐닝 → 요약 카드 | 하단 슬라이드업 | 300ms | ease-out |
| 요약 → 확장 패널 | 패널 높이 확장 + 카메라 블러 | 250ms | spring(1, 80, 10) |
| 에이전트 전환 | 아이콘 크로스페이드 + 라벨 슬라이드 | 500ms | ease-in-out |
| 로딩 → 슬라이더 | 크로스페이드 | 400ms | ease-out |
| 토스트 표시 | 상단 슬라이드 다운 | 200ms in, 3s hold, 200ms out | ease-out / ease-in |
| Discovery 카드 | 순차 페이드인 (stagger) | 각 100ms | ease-out |
| 카드 → 상세 뷰 | Shared element transition | 300ms | ease-in-out |
| 패널 닫기 | 하단 슬라이드 다운 | 200ms | ease-in |
